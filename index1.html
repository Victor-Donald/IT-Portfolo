Overview

This project was executed within a simulated university environment to diagnose and resolve network connectivity failures impacting administrative and student registration systems. The goal was to restore reliable access to critical services while maintaining strict SLA requirements.

Technical Objectives

  • Identify root causes of DHCP assignment failures and APIPA address issues.
  • Restore network connectivity for affected endpoints under tight SLA guidelines.
  • Verify secure database access and network stability through loopback testing.
  • Document diagnostic steps to support repeatable troubleshooting workflows.

Execution

I used CLI tools, specifically ipconfig, to analyze DHCP leases and detect endpoints that had fallen back to APIPA addresses. This allowed me to quickly isolate misconfigured or non-responsive DHCP services.

After identifying the issue, I coordinated with the infrastructure team to perform targeted switch port resets and configuration checks. I then validated connectivity using local loopback tests to ensure that endpoints could securely reach database systems and application servers.

All steps were documented to create a clear troubleshooting path that could be reused for future incidents, supporting SLA compliance and faster resolution times.

Outcome

  • Restored full network connectivity for impacted administrative and student endpoints.
  • Verified secure database access via loopback testing and connectivity checks.
  • Maintained SLA compliance through rapid diagnosis and remediation.
  • Established a repeatable diagnostic workflow for DHCP and APIPA-related issues.
